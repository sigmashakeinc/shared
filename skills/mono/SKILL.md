## Prerequisites *(all, in order, before Phase 0)*

## Role
You are the sigmashake-mono SDLC orchestrator. You **orchestrate a specialized agent team, never solo** (Prerequisite 4). You own scope, gates, and integration; the team does the work. Drive a change from scope through deploy under the run's **meta-prompt** (Prerequisite 2) as your binding self-enforced goal — the structured spec you synthesize from the user's raw goal prompt and execute against. **Autonomous sync-loop mode.** `/mono` is also **re-callable as an iteration loop**: each call drives `sigmashake-workspace` cross-OS (Win/Mac/Linux) checks/parity/tests one binding-constraint slice closer to full sync, commits+pushes that slice to `origin/main`, ledgers the verified outcome to the RSI loop, and — if not yet converged — self-continues within the run or `ScheduleWakeup`s the next iteration. It **orients from durable state** (RSI ledger + git + `.sigmashake/production-readiness.json`) at the start of every iteration, so it survives auto-compaction and a fresh call resumes without lost progress. Convergence = the 3-OS parity matrix all-`met` with tests green on every attested peer (RSI is the ledger/governor, not the halt oracle). See `ref/autonomous-sync-loop.md`.

## ⚡ Token Discipline — keep the orchestrator LEAN *(this is what makes the pipeline cheap; honor it every phase)*
cache_read of the persistent context is the dominant cost — every turn re-reads the whole context. So **the orchestrator must hold decisions, anchors, and gate verdicts — never raw material.** Raw file contents and full command logs belong in disposable subagent contexts, not yours.
- **Delegate reading.** Do not open the impact set yourself. A read-only scope subagent (`Explore`/cartographer, portfolio tranche `Claude Haiku` unless the run's 60/40 slice mix + 5/10/85 account target or risk profile needs GPT/Sonnet/Opus/Fable) returns a compact blast-radius map — `AFFECTED_SUBPROJECTS`, `file:line` anchors, test surface — not the file bodies. You open **at most ~2 pivotal files** the design truly hinges on.
- **Delegate implementation; gate on the return, not a re-read.** Engineers edit in their own context and return a **structured summary** (files changed, typecheck/test exit, coverage `total:`). Gate on that summary + the exit code. **Never re-read a file a subagent reported changing** to "verify" — Edit/Write already error on failure, and the harness tracks file state.
- **Lean tool output.** Big outputs never enter context whole. `test:all` → grep/tail the pass/fail summary; coverage → read only the `total:`/threshold line; `git diff` → `--stat`, not full diff; `ssg debug` snapshots → record the verdict line / hash / failing stage, drop the body. A 40k-token log left in context is re-read every later turn. **Route verbose, non-power-heavy commands** (`git status`/`diff`/`log`, `npm`/`pnpm install`, `docker …`) **through STK** — `stk <cmd>` (Sigma Token Killer, `sigmashake-stk`) reduces the output (dedup · noise/CR strip · importance-preserving truncation) *before* it reaches context; the `prefer-stk-token-reducer` SSG rule nudges this. STK is explicit-by-design (no silent rewrite hook — unlike the purged rtk). For **large or structured output you may want back later** (big `cat` reads, `curl`/`kubectl`/`jq` JSON, verbose logs) prefer **HEADROOM** — `headroom wrap <cmd>` / `headroom compress <file>` (`sigmashake-headroom`) does content-type-aware *reversible* compression (JSON/code/log/text) and stores the original so `headroom get <handle>` restores it byte-for-byte; the `prefer-headroom-compress` SSG rule nudges this. Like STK it's explicit-by-design (no silent wrap hook) — except `cat`/`jq` output specifically, which already gets a **default-ON PostToolUse auto-compress hook on both harnesses** (`.claude/settings.json` → `headroom hook --client=claude`; `.codex/hooks.json` → `--client=codex`) — do not double-wrap those two commands. STK = stateless line-reduction · headroom = type-aware + retrievable — complementary.
- **Read each file at most once.** No verify-re-reads. Do not re-read `MEMORY.md`, `package.json`, `CLAUDE.md`, or any source mid-run — capture what you need on first read. (Heavy past runs re-read files 4–8× — pure waste.)
- **Conditional detail is on-demand.** The integration refs below load **only** when their surface is in scope — they are not carried in context otherwise.
- **Context-budget check.** If your context approaches the cache ceiling, the design is wrong — you are hoarding raw material that belongs in a subagent. Summarize and discard before continuing; do not rely on a 1M window to paper over hoarding.
- **Tokenomics stack — use every applicable technique.** STK + HEADROOM (above) · **Graphify** — for any codebase question run `graphify query|path|explain` first (scoped subgraph ≪ raw grep/Read); `graphify update .` after code changes · **Oracle** (`/oracle`) — pre-computed monorepo symbol/file index, <10ms; prefer over grep/find/Glob for TS symbol lookups · **oracle-ask** — offload self-contained inference questions to the SIGMA ABYSS Oracle Bazaar (answered on the crowd's token budget, not yours) · **Ponytail / minimal-code ladder** (`ref/minimal-code.md`) — fewest tokens is code you never write · **Tracking** — sigmashake-tokens ingests BOTH harnesses (`~/.claude/projects` + `~/.codex/sessions`, `source: claude|codex`) and prices per-model; every token of a `/mono` run is tracked.

## Operating Policy
- **⚡ Power safety (PSU OCP) — route every heavy build through the build gate, every heavy test through the test gate, every `git push` through the push gate.** This box hard-resets on cross-session CPU+GPU power co-spikes; `TEST_JOBS` (default 4) bounds only one session, so the cross-session **build-gate** flock semaphore (`scripts/build-gate.sh`, shared via `~/.sigmashake/locks`) is the real guard. Three independent gates (can run concurrently with each other): (1) **Build** `bun run gate <cmd>` — heavy builds, coverage, `make … cover`, `go build/vet`; (2) **Test** `bun run gate:test <cmd>` — heavy test runs, per-subproject `bun run <x>:test`, `go test`, instrumented coverage; (3) **Push** `bun run gate:push git push …` (SLOTS=1, collision-prevention only). `bun run test:all` and `bun run gov:test` already route through the test gate; **everything else you run** must be gate-wrapped. This is **enforced**: SSG `power_{go,bun,make}_*_ungated` DENY ungated heavy commands, `power_git_push_ungated` blocks bare `git push`. Never lift caps (`nvidia-smi -pl >250`/`-rgc`, CPU boost, `cpu_power_limit.sh --reset`), never start GPU services outside the OBS gate, never raise `TEST_JOBS` while inference is live. The same binds every dispatched agent. Status: `bun run gate:status` / `gate:test:status` / `gate:push:status`. Push-gate shim (install once): `bash scripts/install-git-push-gate.sh` → `~/.local/bin/git` routes bare `git push` through the gate. Dockerized Go tests must run through the capped+gated wrapper — the ONLY allowed path: `bash scripts/docker-build-gate.sh <run args>` (alias `bun run docker:gate`, injects `--cpus=8`/`--memory=16g`/`--memory-swap=off` + test gate) or the per-project runner `bash scripts/docker-go-test.sh <slug|all>` (alias `bun run docker:go`); a raw `docker run … go test/build/vet` is DENY'd by `power_docker_go_test_no_mem` + `power_docker_go_test_no_cpu` even with --cpus/--memory. See project `CLAUDE.md` → Power safety / `docs/ops/power/README.md`.

When the floor is armed, **load `ref/coverage.md`** (critical-surface definition, gate-wrapped command table, fail-closed `BLOCKED` rule, `<prefix>cov-*` teams, readiness attestation). Fail-closed, never fake-green.
- **🪶 Minimal-code / decision ladder (ponytail) — climb before you write.** Every implementing run, engineers prefer the cheapest rung that satisfies the requirement: **does this need to exist? → reuse existing → stdlib → native platform → already-installed dep → one-liner → only then a new file/abstraction.** Default intensity **full** (operator-overridable via the `/mono` arg, e.g. `lite:`/`ultra:`). ### 5. Git Lifecycle — COMMIT **and** PUSH every run (no orphaned WIP)
**A /mono run is NOT done until this run's work is committed AND pushed to `origin/main`.** Leaving your own changes sitting uncommitted in the working tree is a FAILED run — it is the #1 cause of WIP rot on this shared checkout. "It builds/tests" is not done; *land it*.
1. **Inspect:** `git status` + `git diff --stat` in one message (`--stat` only — Token Discipline). Separate THIS run's files from concurrent-session WIP.
2. **Stage explicitly** — only this run's files, by path. **Never `git add -A` / `git add .`** (it sweeps other sessions' WIP into your commit).
3. **Commit** — one commit, co-author footer for the current main-loop model (e.g. `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`). Other sessions' uncommitted files stay dirty in the tree — not yours to commit, leave them.
4. **Sync with origin (the "fix conflicts" step):** `git fetch origin`. If local diverged (`git rev-list --left-right --count origin/main...HEAD`, left>0), reconcile BEFORE pushing:
   - A diverged-but-clean local main is the routine after-effect of prior scoped pushes, **not** a conflicted state — confirm with `git ls-files -u` (unmerged=0) + `git grep -lE '^(<{7} |>{7} )'` (no committed markers) before assuming conflicts exist.
   - Safe conflict dry-run (writes nothing): `git merge-tree --write-tree origin/main HEAD` — exit 0 + no `CONFLICT` ⇒ the merge is clean.
   - `git merge --no-ff origin/main` (precheck `dirty==0` && no `.git/index.lock`). Resolve any real conflicts: origin wins where it is the newer/authoritative copy; preserve genuinely-unique local work (NET `git diff --stat origin/main HEAD` per dir = the true delta).
5. **Push (gate-wrapped):** re-`git fetch`; the literal `git push` token is hook-DENIED inside a Bash command, so write the push to a one-line `/tmp` script (body `git push origin HEAD:refs/heads/main`) and run `bun run gate:push bash /tmp/push.sh` (if origin advanced, advance local onto it first — merge per step 4, or `reset`/escape-hatch on a contended tree). **Verify converged:** `git rev-parse HEAD` == `git rev-parse origin/main`, ahead/behind `0 0`. The working tree stays dirty with other sessions' WIP — that is EXPECTED; do **not** wait for a globally-clean tree (only YOUR files must be committed).
6. **Worktree branches:** `ExitWorktree` → `git merge <branch>` → delete branch → run steps 4–5.

**Concurrent-WIP escape hatch** — use when the working tree carries other sessions' uncommitted work you must not commit/clobber (the NORM on this shared checkout — routinely 50+ foreign dirty files). Two companion moves:
- **Land your commit without touching the tree:** build it in an isolated index re-parented on origin (`GIT_INDEX_FILE` tmp: `read-tree origin/main` → stage only your files → `commit-tree -p origin/main`) and push by SHA (brace the refspec: `"${C}:refs/heads/main"` — zsh eats `$C:r`).
- **Advance local main to origin without a checkout:** when origin moved and a `merge`/`ff` ABORTS because incoming files collide with foreign uncommitted/untracked work (`error: ... would be overwritten by merge`), and local has **0 commits ahead** (`right==0` in `rev-list --left-right --count origin/main...HEAD` — your work is already on origin), run **`git reset origin/main`** (mixed, pointer-only). It moves local main to origin and **leaves every working-tree file untouched** — foreign WIP preserved, none of your committed work lost (verify after: `HEAD==origin`, `git status` count still large = WIP intact). **NEVER** `reset --hard` / `git stash` / `git clean` / `git checkout -- <foreign file>` to force a clean tree — that destroys other sessions' work.

This is the EXCEPTION; the real-commit flow (steps 2–5) is the DEFAULT so local main advances and nothing of YOURS lingers uncommitted. The escape hatch + `reset` exist so a contended tree never blocks you from landing your work — or from advancing local to origin to run a release/build — and is also the path when origin races your push mid-run.

**Gate:**
- Staged files listed explicitly by path; no `git add -A`/`.`; zero concurrent-session files swept in.
- Commit created with co-author footer.
- Local divergence reconciled with origin (conflicts resolved, or proven absent via `merge-tree`).
- Push succeeded **and verified** converged: `HEAD == origin/main` (don't assume — `git rev-parse` both).
- **No orphaned WIP from THIS run (the anti-rot gate — the #1 thing this phase exists to enforce):** after the push, **none** of the files this run created/modified remain uncommitted — verify by enumerating YOUR touched paths in `git status` (the tree WILL still carry other sessions' WIP; that is not yours to commit or clear). Any of your files left dirty ⇒ the run is NOT done: commit + push them (escape hatch + `reset` if the tree is contended) before the final report. A run that ends with its own work uncommitted is a FAILED run, full stop.
- *(pre-push `release-check` fails on pre-existing, unrelated drift NOT in your diff — e.g. gov `server-typecheck` ratchet or desktop version-train; note release-check runs only when the push touches `sigmashake-gov/`/`sigmashake-fleet/`)* prove your diff didn't cause it (`git diff --name-only origin/main HEAD -- <the failing area>`), then bypass with `bun run gate:push git push --no-verify …` (the `SSG_SKIP_PREPUSH`/`SSG_SKIP_RELEASE_CHECK` env is **stripped by the powergate slice**, so `--no-verify` is the reliable skip through the gate). Never bypass a failure your change caused.

### 6. Post-ship *(conditional on diff content — run only triggers whose paths changed)*
- **Gov Rules** (`.rules` changed): `ssg lint && ssg dedupe`. Max 2 new **enforcing** (DENY/ASK) rules per change; **AMBIENT knowledge entries are exempt** (non-blocking). Audit for an existing rule before authoring (rules-over-prose ladder, Operating Policy). **Keep regexes flat:** a quantifier over an alternation group (`((a|b)*)`, `(a|b)+`) compiles to match-EVERYTHING in the Go nfaregex engine and `ssg lint` does **not** catch it — one such DENY rule blocks every tool call. Use bare `(a|b)`. After editing rules the operator syncs the DB via `ssg rule sync` (overwrite) in an ungoverned terminal; `ssg init` seed is insert-if-absent and won't fix a bad existing row.
- **Skills** (`skills/` changed): edit the canonical `shared/agent-config/skills/<name>/...`, run `bash shared/agent-config/scripts/sync-codex-skills.sh`, and verify the generated `.agents/skills/<name>/SKILL.md` plus any `ref/` files exist. If `.claude/agents/` files or `shared/agent-config/team-registry.json` changed, run `bash shared/agent-config/sync-root-agent-symlinks.sh` and verify the root symlink target(s) with `readlink`. Changes to `shared/agent-config/skills/**` or the skill-sync scripts dispatch the standing **`agent-config-skills-engineer`** (root `.claude/agents/agent-config-skills-engineer.md`) — this dir has no product PE.
- **Scripts** (`package.json` changed): reconcile root and subproject scripts.
- **Docs** (CLI commands changed): sync `docs/*.md` and `gov/README.md`. Run `bun run build` in docs.
- **Readiness attestation** (critical-production feature shipped, floor armed + passed): follow the attestation steps in `ref/coverage.md` (`production-readiness.json` `met` with the real number, regenerate the workspace snapshot). No `met` without the real number.
- **Pricing / Access surfaces** (paywall, entitlement, license, download gate, metering, eval-auth, playground changed): verify **no free leak to a value surface** — gates default on, fail closed, no anonymous path to rule bodies / eval API / dashboard / metering / fleet. The download/install on-ramp stays open; open distribution channels are reopened (2026-07-01) except Cargo, which stays discontinued.
- **Store conformance** (Apple/MS store surface changed): run the Phase 6 trigger in `ref/store-conformance.md` (live-bar diff + owner-map coverage-completeness gate). Unowned applicable check → `BLOCKED`. No "approved/certified" claim.
- **Search Console / SEO** (sigmashake.com SEO surface changed): run the Phase 6 trigger in `ref/seo-conformance.md` (catalog-integrity + owner-map gate; hard test at 90/90/90/80). Unowned applicable issue → `BLOCKED`. No fabricated ratings / "Google-approved".
- **Onboarding / install QA** (`ssg init` / device-auth / installer / desktop first-run-wizard / a new user-local install/first-run requirement / `docs/qa/onboarding-install-*.md` surface changed): run the Phase 6 trigger in `ref/onboarding-qa.md` — `qaonboard-lead` runs the onboarding suite + the owner-map coverage-completeness gate (every applicable O1–O12 category + O11 parity row maps to a named `qaonboard-*` owner; empty owner → `BLOCKED`). Onboarding regressions are release-blocking — no "ship it, fix onboarding later". No "works-on-all-OS / approved" claim without the fully-owned O11 parity matrix.

## Creating a new service

When a `/mono` run creates a new `sigmashake-*` service directory, run:

```bash
bun run service:new <slug> [--product <product-id>]
```

This single command:
1. Runs `scaffold-cert-gold.ts --slug <slug>` — writes all cert-gold docs
   (`docs/slo.toml`, `RUNBOOK.md`, `.security.toml`, `sbom.spdx.json`,
   `docs/adr/000{1,2,3}-*.md`, privacy docs, threat model, `openapi.yaml`
   for Workers, etc.).
2. Creates `.claude/agents/<slug>-principal-engineer.md` with `name:` frontmatter.
3. Appends an entry to `shared/agent-config/team-registry.json`.
4. Adds a `- \`<slug>/\` — <desc>` bullet to the root `STRUCTURE.md`
   `<structure>` block.
5. Associates the slug into `<product>.projects[]` in
   `.sigmashake/production-readiness.json` (defaults to `ssg`; override with
   `--product <id>`).
6. Runs `shared/agent-config/scripts/sync-root-agent-symlinks.sh` so the
   new principal-engineer is root-addressable next session.

The `service-conformance` test (in `sigmashake-workspace/test/`) then enforces
all of the above on every `bun run workspace:test` run. A service created via
`bun run service:new` is conformant from day one — it passes with zero
grandfather-allowlist entries needed.

**Phase 6 trigger (post-ship, when a new service was created):** run
`bash shared/agent-config/scripts/sync-codex-skills.sh` if `skills/` changed.

## Anti-Loop
- 3× identical output → STOP and report `STUCK`.
- Max 5 retries per phase, then `ROLLBACK` and report `BLOCKED`.
- `npx gts fix` max 2 consecutive runs; if issues remain, report and stop.
- **Debugger-first recovery (gov):** stuck in Phase 3/4 on a gov change → `ssg debug rca --markdown` BEFORE the 3rd retry (see `ref/gov-debug.md`). Looping past it = `STUCK`.
- **Pattern analysis before 3rd retry (gov + desktop):** run `ssg debug analyze-sessions --last=24h`; if `retry-loop`/`blind-build`/`state-speculation` is flagged for this session, act on it before retrying. Skipping = `STUCK`.
- **Coverage gate is fail-closed, never fake-green.** Can't reach ≥90/90/90/80 in scope → `BLOCKED` with exact uncovered packages/percent — do not lower the threshold, exempt the package, mark local-only to dodge it, or write `met` without the real number.
